# WorkSolo

![WorkSolo Logo](/D:/project/codex/workbook/WorkClosure/Assets/worksolo-mark.png)

WorkSolo 是一个面向个人使用的 Windows 轻量桌面应用，用来记录工作事项、持续推进处理过程，并按时间和项目维度做复盘。

当前仓库对应 `v1.1.0` 开发版源码。

## 产品定位

WorkSolo 聚焦三件事：

- 把个人工作事项从“记下来”推进到“闭环”
- 把推进过程记录下来，方便后续复盘和汇报
- 保持本地保存、启动轻量、学习成本低

## 当前功能

- 工作台
  - 聚焦待推进、超期、异常/阻塞、最近更新
  - 支持点击卡片或事项，直接进入事项详情
- 事项清单
  - 支持新增、编辑、删除、状态流转、搜索、筛选
  - 右侧详情栏分为“基本信息 / 进展记录”两个标签页
- 事项进展记录
  - 支持按日期记录进展、问题、下一步
  - 支持“需跟进”和“关键节点”标记
- 项目分类
  - 支持新增、编辑、删除、启用/停用
  - 明确区分“新建项目”和“编辑项目”
- 总结复盘
  - 支持按日 / 周 / 月查看汇总
  - 支持从统计卡和项目快照继续下钻到具体事项
- 设置与更新
  - 支持主题、默认启动页、提醒开关
  - 支持查看版本信息和更新历史
- 本地数据
  - 正式数据保存在 `AppData\Local\WorkSolo\data.json`
  - 新版首次启动会尝试迁移旧版数据
  - 保存时会自动保留最近备份

## 技术栈

- `.NET 8`
- `WinUI 3`
- `Windows App SDK`
- 本地 `JSON` 持久化

## 目录说明

- `Pages/`：页面视图
- `ViewModels/`：页面状态与交互逻辑
- `Models/`：数据模型
- `Services/`：状态管理、存储、统计逻辑
- `Styles/`：主题和样式资源
- `Assets/`：图标与界面资源
- `Tools/`：辅助启动器源码
- `使用说明.md`：中文使用手册

## 本地运行

先在本地构建：

```powershell
dotnet build .\WorkClosure.csproj -p:Platform=x64
```

构建完成后可从输出目录启动：

```text
bin\x64\Debug\net8.0-windows10.0.19041.0\WorkSolo.exe
```

如果你本地额外保留了根目录 `WorkSolo.exe` 或 `AppLive\WorkSolo.exe`，那属于本地运行产物，不是源码仓库默认内容。

## 数据说明

- 正式数据位置：`C:\Users\你的用户名\AppData\Local\WorkSolo\data.json`
- 旧版本如果把数据保存在项目目录下的 `data\data.json`，新版首次启动时会自动迁移
- 保存前会自动在 `AppData\Local\WorkSolo\backups\` 下保留最近几份备份
- 后续升级时，只要保留这个固定数据目录，原有数据就会继续可用
