# WorkSolo

![WorkSolo Logo](Assets/worksolo-mark.png)

WorkSolo 是一个面向个人使用的 Windows 轻量桌面应用，用来记录工作事项、推进处理过程、标记完成状态，并按时间和项目维度做闭环复盘。

当前仓库为 `V1` 首个正式版本源码仓库。

## 产品定位

WorkSolo 聚焦一件事：

- 让个人工作事项从“记下来”走到“处理完”
- 让每天、每周、每月都能快速复盘
- 保持本地保存、轻量启动、低学习成本

## 当前功能

- 工作台：查看今天到期、已逾期、最近待推进事项
- 事项清单：支持新增、编辑、删除、状态流转、搜索和筛选
- 项目分类：支持新增、编辑、删除和启用停用
- 总结复盘：按日、周、月统计事项推进情况
- 设置页：主题、默认启动页、提醒开关、版本信息
- 本地数据：固定保存在 `AppData\Local\WorkSolo\data.json`
- 自动迁移：新版首次启动时会自动尝试迁移旧版数据
- 自动备份：保存前会保留最近几份数据备份

## 技术栈

- `.NET 8`
- `WinUI 3`
- `Windows App SDK`
- 本地 `JSON` 持久化

## 目录说明

- `Pages/`：页面视图
- `ViewModels/`：页面状态与交互逻辑
- `Models/`：数据模型
- `Services/`：状态管理、存储、统计逻辑
- `Styles/`：主题和样式资源
- `Assets/`：图标与界面资源
- `Tools/`：辅助启动器源码
- `使用说明.md`：项目内中文使用手册

- 不提交根目录 `WorkSolo.exe`
- 不提交 `AppLive/` 运行目录

如果需要给其他人直接双击运行，建议单独打包发布可执行文件，或者使用 GitHub Releases 发布。

## 本地运行

克隆源码后，请先在本地构建：

```powershell
dotnet build .\WorkClosure.csproj -p:Platform=x64
```

构建完成后，可从输出目录启动：

```text
bin\x64\Debug\net8.0-windows10.0.19041.0\WorkSolo.exe
```

如果你在自己的本地开发目录里额外保留了根目录 `WorkSolo.exe` 或 `AppLive\WorkSolo.exe`，那是本地运行产物，不属于源码仓库默认内容。

## 数据说明

- 当前正式数据位置：`C:\Users\你的用户名\AppData\Local\WorkSolo\data.json`
- 历史版本如果把数据保存在项目目录下的 `data\data.json`，新版首次启动时会自动迁移
- 保存前会自动在 `AppData\Local\WorkSolo\backups\` 下保留最近几份备份
- 以后升级新版本时，只要保留这个固定数据目录，原来的数据就会继续可用
