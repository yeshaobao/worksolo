# WorkSolo 使用说明

## 1. 启动方式

- 如果你使用的是我本地开发目录中的完整运行版，可以双击 `Run-WorkSolo.cmd`
- 也可以直接打开 `AppLive\WorkSolo.exe`
- 如果你是从 GitHub 克隆的源码仓库，默认不包含这两个运行产物，需要先本地构建

源码仓库构建命令：

```powershell
dotnet build .\WorkClosure.csproj -p:Platform=x64
```

构建完成后可从输出目录启动：

```text
bin\x64\Debug\net8.0-windows10.0.19041.0\WorkSolo.exe
```

## 2. 数据存放位置

- 所有本地数据都保存在项目根目录下的 `data` 文件夹
- 当前数据文件路径：`data\data.json`

如果系统里存在旧版本数据，程序首次启动时会自动迁移到这里。

## 3. 基本使用流程

### 3.1 快速新增事项

1. 打开“工作台”或“事项清单”
2. 在“快速新增”输入框里填入标题
3. 点击“加入清单”
4. 事项会立即保存到本地，截止日期默认是当天

适合先快速记下工作，再到右侧补充项目、状态和异常说明。

### 3.2 编辑或删除事项

1. 进入“事项清单”
2. 在左侧列表中单击一条事项
3. 在右侧编辑区修改标题、项目、状态、异常标记、截止日期或说明
4. 点击“保存事项”完成更新
5. 如需删除，点击顶部或右侧的“删除事项”

### 3.3 事项状态流转

在“事项清单”顶部可以直接对选中事项执行：

- `开始`
- `完成`
- `取消`

事项标记为完成后会自动记录完成时间。

### 3.4 新建、编辑或删除项目

1. 进入“项目分类”
2. 点击“新建项目”创建空白项目
3. 选中左侧项目后，可在右侧直接编辑名称、说明和启用状态
4. 点击“保存项目”保存修改
5. 如需删除，点击“删除项目”

删除项目后，原来归属该项目的事项会自动转为“未分类”。

### 3.5 查看总结复盘

进入“总结复盘”后可以：

- 按日、周、月切换周期
- 查看新增事项数、完成事项数、未闭环数、逾期数
- 查看项目维度的完成情况
- 查看周期内的完成清单和待继续跟进事项

### 3.6 调整设置

进入“设置”后可以：

- 切换主题模式：跟随系统 / 浅色 / 深色
- 设置默认启动页
- 开关到期提醒
- 查看版本信息
- 打开数据目录
- 打开本说明文档

## 4. 页面说明

### 工作台

- 用来快速新增事项
- 查看今天到期、已逾期、最近待推进的事项
- 适合每天高频打开

### 事项清单

- 用来集中管理所有事项
- 支持搜索、筛选、编辑、状态切换和删除
- 是日常处理事项的主页面

### 项目分类

- 用来维护项目
- 方便后续汇总和筛选

### 总结复盘

- 用来按时间和项目查看工作推进与闭环情况
- 适合周总结、月总结时使用

### 设置

- 用来管理界面偏好和默认行为
- 适合首次使用时做个性化配置

## 5. 使用建议

- 先用“快速新增”记录标题，避免遗漏
- 每天优先处理“已逾期”和“今天到期”
- 每周至少看一次“总结复盘”
- 项目名称尽量固定，避免同类事项分散
