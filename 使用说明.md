# WorkSolo 使用说明

## 1. 启动方式

- 如果你使用的是本地完整运行版，可以双击 `Run-WorkSolo.cmd`
- 也可以直接打开 `AppLive\WorkSolo.exe`
- 如果你是从 GitHub 克隆的源码仓库，默认不包含这两个运行产物，需要先本地构建

源码仓库构建命令：

```powershell
dotnet build .\WorkClosure.csproj -p:Platform=x64
```

构建完成后可从输出目录启动：

```text
bin\x64\Debug\net8.0-windows10.0.19041.0\WorkSolo.exe
```

## 2. 数据存放位置

- 当前正式数据目录：`C:\Users\你的用户名\AppData\Local\WorkSolo`
- 当前数据文件：`C:\Users\你的用户名\AppData\Local\WorkSolo\data.json`
- 自动备份目录：`C:\Users\你的用户名\AppData\Local\WorkSolo\backups`

如果你之前把数据保存在旧版目录里，例如：

- 项目目录下的 `data\data.json`
- 旧版程序目录下的 `data\data.json`

新版首次启动时会自动尝试迁移到新的固定目录。

## 3. 后续更新怎么保留数据

现在版本升级已经改成“程序和数据分离”的方式：

1. 下载新版程序
2. 覆盖或替换旧程序文件
3. 保留 `AppData\Local\WorkSolo` 不动
4. 直接启动新版

这样就可以继续使用原来的数据，不需要每次手动拷贝 `data` 文件夹。

## 4. 基本使用流程

### 4.1 快速新增事项

1. 打开“工作台”或“事项清单”
2. 在“快速新增”输入框里填入标题
3. 点击“加入清单”
4. 事项会立即保存到本地，截止日期默认是当天

适合先快速记下工作，再到右侧补充项目、状态和异常说明。

### 4.2 编辑或删除事项

1. 进入“事项清单”
2. 在左侧列表中单击一条事项
3. 在右侧编辑区修改标题、项目、状态、异常标记、截止日期或说明
4. 点击“保存事项”完成更新
5. 如需删除，点击顶部或右侧的“删除事项”

### 4.3 事项状态流转

在“事项清单”顶部可以直接对选中事项执行：

- `开始`
- `完成`
- `取消`

事项标记为完成后会自动记录完成时间。

### 4.4 新建、编辑或删除项目

1. 进入“项目分类”
2. 点击“新建项目”创建空白项目
3. 选中左侧项目后，可在右侧直接编辑名称、说明和启用状态
4. 点击“保存项目”保存修改
5. 如需删除，点击“删除项目”

删除项目后，原来归属该项目的事项会自动转为“未分类”。

### 4.5 查看总结复盘

进入“总结复盘”后可以：

- 按日、周、月切换周期
- 查看新增事项数、完成事项数、未闭环数、逾期数
- 查看项目维度的完成情况
- 查看周期内的完成清单和待继续跟进事项

### 4.6 调整设置

进入“设置”后可以：

- 切换主题模式：跟随系统 / 浅色 / 深色
- 设置默认启动页
- 开关到期提醒
- 查看版本信息
- 查看数据文件路径

### 4.7 检查更新 / 查看版本历史

进入左侧导航的“检查更新”后可以：

- 查看当前安装版本
- 在线检查 GitHub 最新发布版本
- 直接打开发布页下载更新包
- 查看 `v1.0.1` 与 `v1.0.0` 的版本历史

## 5. 使用建议

- 先用“快速新增”记录标题，避免遗漏
- 每天优先处理“已逾期”和“今天到期”
- 每周至少看一次“总结复盘”
- 项目名称尽量固定，避免同类事项分散
